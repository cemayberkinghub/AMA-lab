# Troubleshooting and Fallback

The lab is designed to preserve the architecture lesson even when product access or environment setup is imperfect.

## Stop rule

If troubleshooting takes more than 10 live minutes or becomes product support, stop and switch to the fallback path:

> We have enough evidence to preserve the harness lesson. We are switching to the fallback path so architecture learning stays on schedule.

## Common issues

| Issue | First response | Fallback |
|---|---|---|
| M365 Copilot or Agent Builder unavailable | Confirm whether anyone at the table has access | Watch facilitator demonstration or use prepared observations |
| Corporate grounding is unavailable | Record what the harness cannot see or cite | Discuss grounding assumptions and implications |
| Microsoft Learn MCP path is blocked | Treat it as boundary evidence | Capture what the surface cannot call or verify |
| GitHub Copilot CLI unavailable | Pair with a participant who has access | Observe facilitator demo and fill the harness comparison table |
| Custom agents do not appear | Restart or inspect environment state once | Continue with prepared examples and record the state gap |
| `/restart` or `/env` fails | Note what state cannot be confirmed | Continue with a facilitator confirmation example |
| Prompt output is vague | Ask for tradeoffs, red lines, and concrete examples | Mark open questions instead of inventing preferences |

## Participation modes

| Mode | Use when | Rule |
|---|---|---|
| Individual work | Most participants have access | Keep outputs comparable through the exercises |
| Paired work | Access is mixed | Pair by access, not seniority |
| Facilitator demonstration | Access is broadly blocked | Participants still capture harness observations |
| Prepared observation | Live tool behavior is too slow or unreliable | Debrief architectural implications, not product troubleshooting |

## Handoff back to learning

After any fallback, answer:

- What could the surface see?
- What could it not call or verify?
- What did the harness fail to make durable?
- What would the higher-control or future Foundry surface need to preserve?
