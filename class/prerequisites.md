# Prerequisites

Use this checklist before class. If something is unavailable, do not spend the session troubleshooting product setup; use the fallback guidance in [`troubleshooting.md`](troubleshooting.md) and stay focused on the harness lesson.

## Required mindset

- Treat this as an architecture-control lab, not a product feature tour.
- Be ready to compare what each surface makes explicit or leaves implicit.
- Keep human authority visible for safety, compliance, accountability, and customer commitment.
- Do not try to solve the whole customer scenario.

## Recommended access

| Surface | Why it helps | If unavailable |
|---|---|---|
| Microsoft 365 Copilot / Agent Builder-style surface | Practice baseline instructions and observe enterprise grounding in a lower-control harness | Watch the facilitator demo or use prepared observations |
| GitHub Copilot CLI | Inspect shared instructions, custom agents, files, tools, and environment state in a higher-control harness | Pair with someone who has access or observe the demo |
| Local Markdown editor | Save `agent-profile-baseline.md` and notes | Use any notes surface that lets you preserve outputs |
| Repository Markdown files | Follow this class package | Continue from the root `class\` path |

## Local files you may create outside this repository

During the lab, you may create local personal files such as:

- `agent-profile-baseline.md`
- `$HOME\.copilot\copilot-instructions.md`
- `$HOME\.copilot\agents\strategy.agent.md`
- `$HOME\.copilot\agents\cloud-solution-architect.agent.md`

These are participant work products for your environment, not repository deliverables.

## Before the live lab

1. Open [`class\README.md`](README.md) and confirm you can find the lab files.
2. Skim [`references.md`](references.md), especially the core model.
3. Confirm whether you can access M365 Copilot and GitHub Copilot CLI.
4. If access is mixed, plan to pair or observe rather than troubleshooting for more than 10 minutes.
5. Keep a notes surface ready for scenario tensions, harness observations, and Spec -> Plan -> Build slices.

## Good enough readiness

You are ready when you can participate in the discussion, capture observations, and compare harnesses even if one tool surface is demonstrated instead of hands-on.
