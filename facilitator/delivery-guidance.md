# Delivery Guidance

This guidance is visible-safe. It supports facilitation without relying on private notes or hidden assumptions.

## Stance

- This is an architecture-control lab, not a product feature walkthrough.
- The scenario is a pressure test, not the design task at the beginning.
- The core teaching object is the harness: what the model can see, do, remember, call, produce, and hand off.
- Agent OS means portable operating model, not a literal OS, runtime, product feature, or single file.
- Human authority remains explicit for safety, compliance, customer commitment, and accountable decisions.

## Common misconceptions and interventions

| Misconception | Signal | Intervention |
|---|---|---|
| Prompt-only thinking | Participants try to craft one perfect prompt | Ask what context, tools, evidence, memory, or handoff the prompt cannot control |
| Product-demo drift | Questions focus on UI mechanics | Ask what the surface lets the agent see, do, remember, call, produce, or hand off |
| Over-automation | Participants want AI to decide regulated outcomes | Ask which decisions require accountable human authority |
| Implicit human authority | Review or approval is assumed but unnamed | Require a named human-owned decision and escalation point |
| Agent OS as literal OS | Participants ask where the OS runs | Reframe as portable operating model created by harness engineering |
| Single-file Agent OS | Participants point to one instruction file | Ask how identity, context, tools, conventions, memory, and handoffs combine |
| Whole-scenario solution drift | Teams design an end-to-end platform | Return to the smallest meaningful MVP slice and out-of-scope boundaries |

## Debrief prompts

- Prompt: What did the original prompt hide?
- Context: What must be durable, cited, or bounded?
- Harness: What controls changed between product surfaces?
- Agent: What changed when the model gained role, instructions, and tools?
- System boundary: Where does AI stop and accountable business process begin?
- Spec: What did you explicitly include and exclude?
- Plan: What risks, validation, and structure did you choose?
- Tasks: What is the first reviewable slice?
- Final comparison: Which tradeoff matters most after applying the agents to the scenario?

## Safety note

Do not add sensitive participant-specific observations, credentials, private environment details, or hidden answer-key framing. If examples are used, present them as calibration examples after participant attempts, not as secret correct answers.
