# Visible-Safe Run of Show

Audience: senior architects practicing controlled agent-system work.

Through-scenario: **AI-Assisted Estimation and Compliance in Regulated Design Work**.

Core model: `prompt -> context -> harness -> agent -> system boundary`; `agent = model + harness`.

Use this file for live timing. Use [`deck-flow.md`](deck-flow.md) for the
slide/story sequence, presenter notes, and transition language.

| Minutes | Segment | Participant action | Facilitator move | Output or debrief |
|---:|---|---|---|---|
| 0-20 | Open and frame | Name one AI opportunity where control matters | Establish architecture-control stance, not product demo | Shared objective and core model |
| 20-30 | Scenario setup | Read scenario and mark tensions | Keep focus on pressure and stakes | Watch list of risks and authority questions |
| 30-50 | Agent OS Personalization | Run the interview and save `agent-profile-baseline.md` | Enforce one question at a time | Agent Operating Profile v1 |
| 50-60 | Baseline instructions | Refine reusable instructions | Keep reusable rules separate from scenario/tool details | Baseline ready for harness tests |
| 60-75 | M365 Agent Builder + grounding | Apply baseline and observe controls | Show lower-control harness surface | M365 harness observations |
| 75-80 | Learn MCP wall | Observe missing or blocked capability | Make the wall a design signal | Boundary note |
| 80-90 | Break/reset | Capture one insight | Prepare CLI surface | Energy reset |
| 90-100 | CLI shared instructions | Draft shared instructions | Show durable file-backed context | Shared instruction draft |
| 100-108 | Strategy agent | Create or inspect strategy agent | Distinguish shared from role-specific behavior | Strategy role notes |
| 108-116 | CSA agent + Learn MCP | Observe CSA setup and tool access | Highlight model + harness = agent | CSA capability contrast |
| 116-120 | Confirm state | Observe `/restart` and `/env` | Make state visible before use | Environment confirmation |
| 120-150 | Spec -> Plan -> Build | Draft or watch intent, plan, and tasks | Prevent final-solution overreach | Reviewable slices |
| 150-175 | Harness comparison + Foundry projection | Compare surfaces | Convert product observations into architecture tradeoffs | Key tradeoffs and projection |
| 175-180 | Closing reflection | Answer final prompt | Close with human authority | Closing insight |

## Compression rules

- If behind, cut explanation before cutting evidence.
- Preserve the Spec -> Plan -> Build transfer before final comparison.
- Use prepared observations when access fails.
- Keep the closing human-authority prompt.

## Debrief anchor

Ask: **If we succeed, what does this system make easier for humans, and what must it never decide on their behalf?**
